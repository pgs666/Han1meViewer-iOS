## 改动总结

### P1-S3: 搜索重置按钮不再触发搜索
- **文件**: `iosApp/SearchView.swift`
- **改动**: `onReset` 回调只调用 `viewModel.resetFilters()`，移除了 `viewModel.search()` 调用
- **原因**: 点击重置按钮时用户期望只清空筛选条件，而不是立即执行搜索

### P1-S4: 空关键词不再发送请求
- **文件**: `iosApp/SearchView.swift`
- **改动**: `onSubmit` 增加 `guard !trimmed.isEmpty || viewModel.filters.activeCount > 0` 检查
- **原因**: 防止空关键词触发无意义的网络请求

### P1-S5: 搜索结果添加下拉刷新
- **文件**: `iosApp/SearchView.swift`
- **改动**: 给 `resultList` 添加 `.refreshable { viewModel.search(keyword: keyword) }`
- **原因**: 对齐其他页面的刷新交互模式

### P1-D2: 搜索历史去重
- **文件**: `SearchHistory.sq`, `SearchHistoryStore.kt`
- **改动**: 
  - SQL 新增 `deleteByKeyword` 查询
  - `record()` 方法在插入前先删除同关键词记录
- **原因**: 防止重复搜索关键词在历史中堆积

### P1-D1: updateProgress 不污染 watched_at
- **文件**: `WatchHistory.sq`, `WatchHistoryStore.kt`, `VideoFeature.kt`, `VideoDetailViewModel.swift`
- **改动**:
  - SQL 新增 `updateProgress` 查询，只更新 `playback_position_millis`
  - `WatchHistoryStore` 新增 `updateProgress` 方法
  - `VideoFeature.recordPlaybackPosition` 改用 `updateProgress` 而非 `record`
  - Swift 调用方适配新签名（移除 title/coverUrl 参数）
- **原因**: 原实现每次保存进度都更新 `watched_at_epoch_millis`，导致"最近观看"排序被污染
- **参考**: Android `WatchHistoryDao.updateProgress`

### P1-S1: 未登录 guard
- **文件**: `iosApp/VideoDetailViewModel.swift`, `iosApp/Localizable.xcstrings`
- **改动**: `toggleFavorite`/`toggleWatchLater`/`setMyListItem` 开头检查 `snapshot.currentUserId != nil`
- **原因**: 未登录用户点击收藏/稍后看会发送无效请求并收到 419 错误
- **参考**: Android `VideoRouteActions.toggleFavorite` 检查 `Preferences.isIsAlreadyLogin`

### P1-C3: LoginView cookie 导入序列化
- **文件**: `iosApp/LoginView.swift`
- **改动**: 
  - 新增 `importStateQueue` 串行队列
  - 用 `tryBeginImport()`/`finishImport()`/`markResolved()`/`isResolved()` 替换裸 `Bool` 检查
- **原因**: `cookiesDidChange` 可能在后台线程回调，与主线程的 `webView(_:didFinish:)` 存在竞态条件
- **参考**: `CloudflareChallengeView` 的 `importStateQueue` 模式

### P1-C4: currentSessionSnapshot 失败时清 session
- **文件**: `shared/.../auth/WebLoginFeature.kt`
- **改动**: `currentSessionSnapshot()` 的 catch 分支增加 `error.shouldClearSession()` 检查
- **原因**: 当 HTML 解析器检测到登录态过期抛出 `DomainException(Auth)` 时，应清除 session 标记，否则后续请求仍认为已登录
- **参考**: Android `MainViewModel` 在 `LoginStateExpiredException` 时调用 `logout()`
