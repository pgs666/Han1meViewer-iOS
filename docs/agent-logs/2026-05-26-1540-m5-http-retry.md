# M5: HTTP 请求重试机制

## User Input

Original:
M5. 没有 HTTP 重试 — OkHttp 默认 retryOnConnectionFailure(true) 对 idempotent GET 自动重试。Ktor 默认不重试。影响：弱网首页/搜索一抖就 fail。

English translation:
M5. No HTTP retry — OkHttp defaults to retryOnConnectionFailure(true) for idempotent GET requests. Ktor doesn't retry by default. Impact: weak network causes home/search to fail on first hiccup.

## What Changed
- Added `HttpRequestRetry` plugin to `Han1meHttpClient.kt`
- Retries up to 2 times on non-DomainException exceptions (network errors, timeouts, connection failures)
- Uses exponential backoff between retries
- Does NOT retry on DomainException (auth errors, CF blocks, 4xx, 5xx)

## Why Changed
- OkHttp retries connection failures automatically; Ktor does not
- Weak network conditions cause single-request failures that would succeed on retry
- Review item M5 from `review-ios-vs-android.md`

## Known Limits
- Only retries on exceptions, not on 5xx responses (those are thrown as DomainException by the response validator)
- Could add `retryIf` for 5xx if needed in the future
