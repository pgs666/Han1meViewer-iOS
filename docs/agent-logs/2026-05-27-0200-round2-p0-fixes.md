## What Changed

### P0-1: 错误响应 cookie 保存
- `Han1meHttpClient.kt`: `createHan1meHttpClient` 新增 `saveCookies` 回调参数，通过 `ResponseObserver` 插件在每个响应上保存 cookie（早于 `HttpResponseValidator` 抛异常）。
- 所有 8 个 Repository：构造器新增可选 `client: HttpClient? = null` 参数；类体中 `client = client ?: createHan1meHttpClient(cookieBridge::saveResponseCookies)`。
- `SharedAppEnvironment.kt`: 创建共享 `KtorCookieBridge`，传入 httpClient 的 `saveCookies` 回调。
- `sessionStore` 参数改为 `private val` 以确保类体可访问。

### P0-2: WebView User-Agent 一致性
- `LoginView.swift:131`: `webView.customUserAgent = HanimeNetworkDefaults.DEFAULT_USER_AGENT`
- `CloudflareChallengeView.swift:131`: 同上
- 对齐 Android `LoginActivity.kt:108` / `CloudflareActivity.kt:72` 的 `settings.userAgentString = USER_AGENT`

### P0-3: CF 路径 cookie 元数据保留
- `CloudflareFeature.kt`: 新增 `importChallengeCookiesJson` 方法，使用 `WebCookiePayload` 解析 JSON，保留 domain/path/expires/secure/httpOnly。
- `WebLoginFeature.kt`: `WebCookiePayload` 从 `private` 改为 `internal`。
- `CloudflareChallengeView.swift`: `importClearanceCookies` 改用 JSON 编码（`encodeCookiesForImport`）+ `importChallengeCookiesJson`，替代原 cookie header 字符串通路。

### P0-4: HTML 级登录态过期检查
- `HtmlParser.kt`: `parseHome` 新增 `isAlreadyLogin: Boolean = false` 参数。
- `KsoupHtmlParser.kt`: `parseHome` 新增检查 `if (isAlreadyLogin && isLoginStateExpired(...))` → 抛 `DomainException(DomainError.Auth)`。
- `isLoginStateExpired` 逻辑对齐 Android `Parser.kt:203-205`：`userHref.contains("/login") || username.isNullOrBlank()`。
- `KtorHomeRepository.kt`: 从 `sessionStore.loadCookies().hasConfirmedLogin()` 获取 `isAlreadyLogin` 并传入。

## Why Changed
- 第一轮 H4/H6/H7 修复只覆盖了成功路径；P0-1~P0-4 补齐了错误路径、CF 路径、HTML 级别的漏洞。
- Android 的 OkHttp `CookieJar.saveFromResponse` 自动在每个响应（含错误）保存 cookie，iOS 需要显式实现。
- Android 的 WebView 显式设置 UA 与 Ktor 一致，iOS 之前缺失。
- Android 的 `Parser.kt` 有 HTML 级登录态检查，iOS `KsoupHtmlParser` 之前没有。

## Verification
- 本地 JVM 测试通过（`:shared:jvmTest`）。
- CI 待验证。

## Mistakes
- 初始实现用 `observeResponse`（不存在于 Ktor 3.3.2），改为 `ResponseObserver` 插件。
- `isLoginStateExpired` helper 方法因 `private companion object` 匹配失败未写入，后补。
- `by lazy` 在构造器参数中不工作，改为类体内属性。

## Known Limits
- `SharedAppEnvironment` 共享 httpClient 的 `ResponseObserver` 使用 `sharedCookieBridge`，各 Repository 的手动 `saveResponseCookies` 调用保留为冗余安全网。
- `CoroutineScope(Dispatchers.Unconfined).launch` 已移除，改用 `ResponseObserver` 的 suspend 回调。
