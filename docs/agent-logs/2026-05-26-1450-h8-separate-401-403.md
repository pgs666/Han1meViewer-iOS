# H8: Separate 401 (Auth) from 403 (Network/IP block)

Date: 2026-05-26 14:50:00 +08:00

## What Changed

- `Han1meHttpClient`: 401 → `DomainError.Auth`, 403 → `DomainError.Network(403)`.
- `WebLoginFeature.shouldClearSession()`: Only clear on 401, not 403.

## Why

iOS was mapping both 401 and 403 to `Auth`, causing session cookies to be cleared on IP/region blocks (403). After CF challenge is already filtered out, remaining 403s are typically IP bans or region restrictions, not authentication failures. Android distinguishes these with separate exception types.
