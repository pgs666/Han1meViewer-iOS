# 修复 M10 + L8

日期：2026-05-27 07:00 CST

## M10 — WebLogin JS 启发式过宽

### 问题
`LoginView.swift` 的 `evaluateLoginSuccess` 中 JS 检测登录态使用 `[href*="/user"]` 选择器，
会误匹配 `/users`（用户列表页），导致在非登录页面也触发 cookie 导入。

### 修复
将选择器改为 `#user-modal-trigger` + `/\/user\/\d+/` 正则匹配。
对齐 Android `Parser.kt` 的 `getElementById("user-modal-trigger")` + `/user/(\d+)` 正则。

### 技术细节
Swift 多行字符串 `"""..."""` 中的正则需要双重转义：
- `\\/user\\/\\d+` → Swift 产生 `\/user\/\d+` → JavaScript 正则 `/\/user\/\d+/`

### 验证
- CI run 26467016284 通过

## L8 — Background audio modes / PiP 没配 entitlements

### 问题
`Han1meViewer.entitlements` 为空，锁屏后 AVPlayer 停播，不支持 PiP。

### 修复
添加 `UIBackgroundModes` → `audio`。iOS PiP 不需要额外 entitlement，
只需 background audio mode + AVPlayerLayer。
