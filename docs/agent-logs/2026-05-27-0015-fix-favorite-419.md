# 修复详情页收藏按钮 HTTP 419

## User Input

Original:
目前详情页的收藏按钮是坏的，请你参考Android版本进行修复，不要生造东西

English translation:
The favorite button on the detail page is currently broken. Please fix it by referencing the Android version; don't invent something new.

Follow-up original:
点击收藏的时候，弹窗提示：failed to update list state HTTP 419，修复它并推送ci

English translation:
When clicking favorite, the popup says: failed to update list state HTTP 419. Fix it and push CI.

## What Changed
- `CookieHeaderProvider.kt`: De-duplicate cookies by name before building the Cookie header
- Prefer cookies with the longest path, then exact domain (`hanime1.me`) over wildcard domain (`.hanime1.me`)
- `KtorVideoRepository.kt`: Changed favorite mutation error message from generic list-state message to favorite-state message
- `CookieHeaderProviderTest.kt`: Added regression test for duplicate `hanime1_session` cookies

## Why Changed
- Android uses a real CookieJar; duplicate cookies with the same name are resolved by domain/path matching before requests
- iOS manually concatenated all matching stored cookies, so stale wildcard-domain session cookies could be sent alongside fresh exact-domain session cookies
- Laravel CSRF validation can return HTTP 419 when `_token` belongs to one session but the request uses another stale session cookie
- This aligns iOS cookie selection more closely with Android behavior instead of changing the `/like` endpoint fields
